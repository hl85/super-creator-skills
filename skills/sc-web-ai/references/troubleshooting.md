# 常见问题排查

## 浏览器工具相关

### Q: 调用 browser_* 工具提示"找不到工具"
**可能原因：** IDE 没有内置浏览器 MCP，或没有安装 @playwright/mcp

**解决方案：**
1. 确认你使用的是 TRAE/Cursor 等支持 MCP 的 IDE
2. 检查 MCP 配置，参考 `mcp-setup.md`
3. 重启 IDE 让 MCP 配置生效

---

### Q: browser_navigate 报 ERR_CONNECTION_RESET
**可能原因：** 网络问题，无法访问 Gemini/ChatGPT

**解决方案：**
1. 检查系统代理是否开启并正常工作
2. 先在系统浏览器中手动确认能访问 gemini.google.com / chatgpt.com
3. 如果是使用 @playwright/mcp，配置代理服务器参数
4. TRAE 内置浏览器会使用系统代理，确认代理正常即可

---

### Q: 元素点击报错"pointer-events: none"
**可能原因：** 元素还在加载中，或被遮罩层遮挡

**解决方案：**
1. 等待几秒后重试（browser_wait_for(time: 2)）
2. 重新 snapshot 获取最新的元素 ref
3. 检查是否有弹窗/遮罩需要先关闭
4. 点击父元素而不是子元素

---

### Q: browser_type 输入不生效
**可能原因：** 输入框没有真正聚焦

**解决方案：**
1. 先 browser_click 点击输入框，确认 snapshot 显示状态为 [focused]
2. 再调用 browser_type
3. 可以尝试设置 slowly: true 逐字输入
4. 某些输入框需要先清除现有内容，设置 clear: true

---

## Gemini 相关问题

### Q: Gemini 一直显示"正在思考"不出图
**可能原因：** 触发限流、网络波动、提示词问题

**解决方案：**
1. 等待最多 60 秒
2. 如果还没反应，刷新页面（browser_navigate 重新访问当前 URL）
3. 简化提示词，避免敏感内容
4. 新开一个对话重试

---

### Q: 提示"很抱歉，我无法生成图片"
**可能原因：** 提示词触发了内容安全策略

**解决方案：**
1. 修改提示词，避免可能的敏感词
2. 用更中性、描述性的语言
3. 避免提到特定人物、品牌、版权内容
4. 重试一次，有时是误判

---

### Q: 下载按钮点击了但找不到文件
**可能原因：** 浏览器下载到了默认下载目录

**解决方案：**
1. 检查系统默认下载目录（~/Downloads/）
2. 图片文件名通常类似 `gemini-generated-image-*.png`
3. 如果需要移动到工作目录，用文件系统工具移动即可

---

## ChatGPT 相关问题

### Q: ChatGPT 生图很慢
**说明：** DALL-E 3 生成本身就比较慢，通常需要 20-60 秒，这是正常的

**解决方案：**
1. 耐心等待，延长轮询间隔到 5 秒
2. 超时时间设置为 90 秒
3. 如果超过 2 分钟没反应，刷新重试

---

### Q: "更多操作"按钮点不动
**可能原因：** 图片还在加载中，按钮处于 disabled 状态

**解决方案：**
1. 等图片完全显示后再等 3-5 秒
2. 重新 snapshot 确认按钮不再是 disabled
3. 不要在图片刚出现时立刻点击

---

### Q: ChatGPT 提示"你已达到当前使用上限"
**可能原因：** 免费账户有生图额度限制，或 Plus 账户达到了速率限制

**解决方案：**
1. 等待一段时间（通常几小时后恢复）
2. 切换到 Gemini 生图
3. 如果有 API Key，切换到 API 方式生图

---

## 通用问题

### Q: snapshot 返回的元素太多找不到目标
**解决方案：**
1. 使用 `interactive: true` 参数只显示可交互元素
2. 使用 `selector: "css选择器"` 限定范围
3. 使用 `compact: true` 获取更精简的输出
4. 根据 name/role 特征在输出中搜索

---

### Q: 操作后页面没变化
**可能原因：** 点击没生效、页面在加载、操作了错误的元素

**解决方案：**
1. 操作后立即 snapshot 确认状态变化
2. 如果元素 ref 过期了，重新获取新的 ref
3. 检查是否有验证步骤需要完成（如人机验证）
4. 等待几秒给页面响应时间

---

### Q: 出现人机验证（CAPTCHA）
**解决方案：**
1. 不要尝试自动识别验证码
2. 提示用户手动在浏览器中完成验证
3. 验证完成后继续操作
4. 这是正常的网站安全措施

---

## 兜底策略

如果自动操作遇到无法解决的问题：
1. **降级为提示用户手动操作**：告诉用户"请在浏览器中手动点击 XX 按钮，完成后告诉我"
2. **切换平台**：Gemini 不行就试 ChatGPT，反之亦然
3. **切换生图方式**：网页版不行就用 API（如果有 Key）
4. **简化任务**：先生成一张简单的图片确认流程通畅，再生成复杂的
