# notion

SaaS dashboard aesthetic with clean data focus and productivity tool styling

## Atmosphere

The organized clarity of a digital workspace where every pixel serves a purpose — a sanctuary of productivity and structured thought. It feels like the first five minutes of a focused deep-work session, where the interface disappears and only the data remains, framed by the subtle elegance of a well-designed SaaS dashboard. A world of soft grays and crisp borders that whispers efficiency and professional calm.

## Design Aesthetic

Clean, functional SaaS interface aesthetic. Dashboard-inspired layouts with clear data hierarchy. Notion, Linear, and modern productivity tool styling. Information-dense but organized. Professional and trustworthy.

## Background

- Color: Light Gray (#F7F7F5) or Pure White (#FFFFFF)
- Texture: None - clean solid backgrounds

## Typography

### Primary Font (Headlines)

System UI stack or Inter. Semi-bold weight for emphasis. Clean, functional letterforms. Slightly tighter letter-spacing.

### Secondary Font (Body)

Same family in regular weight. Optimized for screen reading. Comfortable line height (1.5-1.6).

## Visual Elements

- Card-based layouts with subtle borders or shadows
- Clean data tables and charts
- Progress bars and metric displays
- Icon-based navigation hints
- Checkbox and toggle styling
- Tag and label chips
- Subtle hover state styling
- Breadcrumb and hierarchy indicators

## Style Rules

### Do

- Use card-based content organization
- Create clear data hierarchy
- Use subtle shadows and borders
- Keep layouts grid-aligned
- Present metrics prominently

### Avoid

- Use decorative illustrations or complex backgrounds — every element must have a functional or data-driven purpose.
- Use rounded, organic, or 'blob' shapes; stick to strict rectangular cards with subtle 4-6px corner radii.
- Add gradients, drop shadows with high blur, or any 'skeuomorphic' effects that break the flat, digital-first aesthetic.
- Create cluttered layouts that obscure the clear hierarchy of information; use generous white space to separate distinct data modules.
