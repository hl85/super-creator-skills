# Content-Mining → Writer 衔接指南

本文档说明如何将 sc-content-mining skill 的 Phase 3 输出，无缝衔接到 sc-writer 的输入流程。

## 概述

sc-content-mining 负责从原始素材（1v1 纪要、课程录音、文章等）中挖掘选题和认知错位；sc-writer 负责将选定的选题落地为平台适配的内容（大纲 + 草稿 + 图片规格）。

本手册定义两者之间的标准衔接方式，确保挖掘输出可以直接被 sc-writer 消费。

## 输入格式：sc-content-mining Phase 3 标准输出

sc-content-mining Phase 3 完成后，输出以下结构化选题清单：

```markdown
## 选题清单（按传播力排序）

### 1. [选题标题] 传播力 ⭐⭐⭐⭐⭐
- 核心主张：一句话概括本文要论证的观点
- 认知错位：
  - 受众以为："..." 
  - 真相："..."
- 来源素材：[素材名称/路径]
- 建议呈现：bold + comparison / chalkboard + list
- 核心金句："..."（可直接用于封面）
- 目标受众：[一句话描述目标读者]
- 适用平台：xhs / wechat

### 2. ...
```

**关键字段说明：**

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 核心主张 | 本文唯一要论证的观点，≤15 字 | ✅ |
| 认知错位 | 受众以为 vs 真相的对比结构 | ✅ |
| 来源素材 | 选题出处，用于回溯验证 | ✅ |
| 核心金句 | 可直接做封面标题的短句 | ✅ |
| 目标受众 | 一句话：谁 + 当前信念 + 看完收获 | ✅ |
| 建议呈现 | 推荐的视觉风格 + 布局组合 | ⭕ |
| 适用平台 | 建议发布的平台 | ⭕ |

## 衔接步骤

### Step 1：选定 Topic

从选题清单中选择一个选题，确认：
- 传播力评分 ≥ ⭐⭐⭐
- 有明确的认知错位结构
- 来源素材可追溯

**选定后，提取以下信息交给 sc-writer：**
- 选题标题
- 核心主张
- 认知错位（以为 vs 真相）
- 来源素材路径
- 核心金句

### Step 2：确认定位（Outline Stage）

进入 sc-writer outline 阶段时，将 mining 输出映射到以下定位要素：

| mining 字段 | → | sc-writer outline 输入 |
|-------------|---|----------------------|
| 核心主张 | → | core claim（全文唯一论点） |
| 受众以为 | → | reader 当前信念 |
| 真相 | → | reader 看完收获 |
| 目标受众 | → | reader 身份描述 |
| 核心金句 | → | 封面主标题候选 |
| 建议呈现 | → | 视觉风格/布局参考 |

### Step 3：调整 Outline

在 outline 阶段，基于 mining 输出补充：
1. **图片系列结构**：根据认知错位设计铺垫→核心→收获的滑动逻辑
2. **素材填充**：从来源素材中提取具体案例、数据、原话
3. **平台适配**：根据目标平台（xhs/wechat）调整颗粒度

### Step 4：进入 Draft

outline 确认后，进入 draft 阶段：
- mining 输出的**认知错位结构** → 正文 Hook + Core 的骨架
- mining 输出的**核心金句** → 封面标题 / 正文首句
- mining 输出的**来源素材** → 正文案例 / 细节补充

## 字段映射表

| sc-content-mining 输出 | → | sc-writer 输入 | 阶段 |
|---------------------|---|---------------|------|
| 核心主张 | → | title / core claim | outline |
| 认知错位.受众以为 | → | reader 当前信念 / setup 页内容 | outline |
| 认知错位.真相 | → | reader 收获 / core 页内容 | outline |
| 核心金句 | → | 封面主标题 / 正文 Hook | outline + draft |
| 来源素材 | → | 正文案例补充 / 真实性验证 | draft |
| 目标受众 | → | reader 定位 | outline |
| 建议呈现 | → | style / layout 参考 | draft |
| 适用平台 | → | platform 参数 | 全程 |

## 📌 示例：面试辅导场景衔接

以下以面试辅导为例，演示完整的衔接流程。

### 原始 mining 输出（节选）

```markdown
### 1. 面试官在赌你潜力，不是考你技术 传播力 ⭐⭐⭐⭐⭐
- 核心主张：面试本质是赌潜力，不是技术考试
- 认知错位：
  - 受众以为："面试=技术考试，技术好就能过"
  - 真相："面试官在赌你未来的成长潜力"
- 来源素材：courses/面试辅导/沈安迪-1v1纪要.md
- 建议呈现：bold + comparison
- 核心金句："你花90%精力准备的，只是入场券"
- 目标受众：3-5年前端求职者，以为技术好就能进大厂
- 适用平台：xhs
```

### 衔接到 sc-writer outline

**Step 1：定位确认**

```
reader: 正在求职的 3-5 年经验前端，以为技术好就能进大厂，看完会知道面试官其实在赌潜力
core claim: 面试本质是赌潜力，不是技术考试
```

**Step 2：封面设计**

```
主标题：你花90%精力准备的，只是入场券
副标题：面试官真正在赌的是这个
visual_description: 入场券燃烧+门后光芒
```

**Step 3：图片系列规划**

| 位置 | 标题 | 要点 | 对应 mining 字段 |
|------|------|------|-----------------|
| 封面 | 你花90%精力准备的，只是入场券 | — | 核心金句 |
| 铺垫 | 你以为的面试 | ❌ 技术考试 ❌ 刷题通关 | 认知错位.受众以为 |
| 核心 | 面试官在赌什么 | ✅ 潜力 ✅ 成长速度 | 认知错位.真相 |
| 收获 | 3个信号让面试官押你 | 1. 复盘能力 2. 学习速度 3. 问题定义 | 延伸补充 |
| 结尾 | 别再死磕技术了 | 收藏备用 | CTA |

### 衔接到 sc-writer draft

**正文结构：**

- **Hook**（来自认知错位）："你有没有这种感觉——明明技术不差，但面试就是过不了"
- **Core**（来自真相 + 素材案例）：用沈安迪的案例说明面试官如何判断潜力
- **CTA**："左滑看 3 个让面试官押你的信号→"

**标签**（来自目标受众 + 内容类型）：
```
#面试 #求职 #程序员面试 #大厂面试 #面试技巧
```

## 常见问题

**Q: mining 输出的选题没有核心金句怎么办？**
A: 在 outline 阶段基于核心主张自行提炼，确保 ≤15 字且有冲击力。

**Q: 来源素材找不到了怎么办？**
A: 优先使用有明确来源的选题。如果素材丢失但选题很好，在 draft 阶段标注"案例为典型化描述"。

**Q: 一个 mining 选题可以适配多个平台吗？**
A: 可以。核心主张不变，调整颗粒度和呈现方式即可——xhs 偏短句+视觉，wechat 偏深度+长文。
